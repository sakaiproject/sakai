Descripción

La mayoría de los sistemas operativos modernos proporcionan soporte para WebDAV, haciendo que los ficheros de un servidor WebDAV aparezcan como almacenados en un directorio local.

WebDAV proporciona funcionalidades para crear, cambiar y mover documentos en un servidor remoto (túpicamente un servidor web). Esto se utiliza sobre todo para permitir la edición de los documentos que sirve un servidor web.

Es posible copiar un archivo de su sistema local a Recursos usando el botón Añadir en la herramienta Recursos. Si quiere copiar varios archivos o una carpeta entera de una sola vez, el software WebDAV se puede usar para arrastrar los archivos o carpetas desde el ordenador local a Recursos de Sakai o viceversa.

Se requieren varios pasos de configuración para que su ordenador trate el sitio de Recursos como otra carpeta de su sistema de archivos local. Una vez ha configurado una carpeta para un sitio en particular usando WebDAV, puede arrastrar archivos y carpetas al área Recursos del sitio desde su ordenador local o viceversa. Tenga en cuenta que estas conexiones son específicas del ordenador que está usando, así que si cambia el/los ordenadores tendrá que configurar la conexión de nuevo.

Una vez configurado su cliente WebDav, se le solicitarán su usuario y contraseña para el acceso a los datos. DEBE INDICAR EL USUARIO Y CONTRASEÑA CON EL QUE ACCEDIÓ A SAKAI.

Configurar WebDAV para Windows XP

Nota: Debe haber al menos un objeto (archivo, carpeta etc.) en el área de recursos del sitio antes de comenzar el proceso de configuración. Este proceso debe realizarse una vez para cada sitio en Sakai antes de que se pueda usar WebDAV para transferir archivos a cualquier sitio, o viceversa.

  1. Doble-click en el icono de Mis Sitios de Red en el escritorio del PC para abrir la carpeta Sitios de Red.
  2. Doble-click en el icono Agregar Sitios de Red.
  3. Hacer click en Siguiente.
  4. Hacer click en la opción: "Elija otra ubicación de red."
  5. Hacer click en Siguiente.
  6. En el campo "Dirección de Red o Internet", introducir la dirección URL para este sitio mostrada arriba.
  7. Hacer click en Siguiente.
    Nota: Si recibe un mensaje de alerta como "La ruta que ha introducido no parece válida," es posible que aún no haya recursos en el sitio. Vaya al sitio y añada un archivo para inicializar la carpeta de Recursos, y vuelva al Paso 1.
  8. Cuando pregunte por el nombre de usuario y password, introducir el nombre de usuario y password de Sakai
  9. Introducir un nombre para los recursos del sitio que se está añadiendo. Este nombre se usa en tu ordenador cuando se referencie la carpeta de recursos del sitio.
  10. Hacer click en "Finalizar" para completar la configuración.
  11. Cuando el proceso de configuración haya terminado, la carpeta de recursos del sitio aparecerá en Mis Sitios de Red como una carpeta. Aparecerá también bajo Mis Sitios de Red en el Explorador de Windows, donde actuará como cualquier otra carpeta dentro del Explorador.
  12. Ahora es posible arrastrar archivos y carpetas al ordenador y la carpeta de Recursos de cada sitio en Sakai, y viceversa.

Configurar WebDAV para Windows Vista / Windows 7

Nota 1: En primer lugar, instale el parche de Webdav para Windows Vista / Windows 7

Nota 2: Debe haber al menos un objeto (archivo, carpeta etc.) en el área de recursos del sitio antes de comenzar el proceso de configuración. Este proceso debe realizarse una vez para cada sitio en Sakai antes de que se pueda usar WebDAV para transferir archivos a cualquier sitio, o viceversa.

Nota 3: Existen unos cuantos problemas conocidos con el soporte de WebDAV en Windows Vista / Windows 7. Puede experimentar dificultades usando WebDAV como se describe a continuación.

  1. Desde el escritorio, hacer click en Equipo.
  2. Hacer click en Conectar a Unidad de Red.
  3. En la pantalla de Conectar a Unidad de Red, hacer click en "Conectarse a un sitio Web para usarlo como almacén de documentos e imágenes", y despues hacer click en Siguiente.
  4. Hacer click en "Elija una ubicación de Red predeterminada", y luego en Siguiente.
  5. En el campo "Dirección de Red o Internet", introducir la dirección URL para este sitio mostrada arriba.
  6. Cuando lo indique, introducir el nombre de usuario y password de Sakai.
  7. Ahora es posible arrastrar archivos y carpetas al ordenador y la carpeta de Recursos de cada sitio en Sakai, o viceversa. Es posible que no se pueda acceder al contenido de la carpeta desde Windows, pero igualmente con el botón derecho se pueden pegar archivos y carpetas.

Configurar WebDAV para Mac (OS 10.0 - 10.3)

Nota 1: Estas instrucciones incluyen la descarga de un programa llamado Goliath. Algunos usuarios han informado de errores usando este programa con Sakai, incluyendo la imposibilidad de crear nuevas carpetas o descargar carpetas a su ordenador local.

Nota 2: Debe haber al menos un objeto (archivo, carpeta etc.) en el área de recursos del sitio antes de comenzar el proceso de configuración. Este proceso debe realizarse una vez para cada sitio en Sakai antes de que se pueda usar WebDAV para transferir archivos a cualquier sitio, o viceversa.

  1. Descargar el cliente de WebDav, Goliath, en:
    http://www.webdav.org/goliath/#download
  2. Descomprimir el archivo y hacer doble-click en el fichero .dmg. (Dependiendo de la versión y configuración de Stuffit, la imagen del disco puede montarse automáticamente.)
  3. Ahora estará disponible una imagen de disco llamada"Goliath 1.0". Arrastrar y soltar la carpeta Goliath contenida dentro de la imagen de disco al disco duro.
  4. Abrir la aplicación Goliath. Se encontrará probablemente en la carpeta Aplicaciones.
  5. En el cuadro de diálogo de Conexión WebDav, introducir la ruta como se indica arriba.
  6. Introducir el nombre de usuario y password de Sakai.
  7. Hacer click en la pestaña de Opciones Avanzadas y comprobar el cuadro junto a "Deshabilitar la Codificación de un Recurso Macintosh."
  8. Hacer click en "OK"
  9. Aparecerá ahora una ventana en la pantalla del Mac que representa los recursos que hay en el sitio. Ahora es posible arrastrar y soltar archivos dentro de esta ventana o usar el menu "Web" para subir o descargar archivos.

Configurar WebDAV para Mac (OS 10.4)

Nota: Debe haber al menos un objeto (archivo, carpeta etc.) en el área de recursos del sitio antes de comenzar el proceso de configuración. Este proceso debe realizarse una vez para cada sitio en Sakai antes de que se pueda usar WebDAV para transferir archivos a cualquier sitio, o viceversa.

  1. Desde el Buscador, seleccionar "Ir" desde la barra de Menu y después seleccionar "Conectar a un Servidor..."
  2. En el cuadro de diálogo resultante, introducir la ruta como se muestra arriba.
  3. Presionar el botón "+" para almacenar esta ruta para su uso futuro.
  4. Hacer click en "Conectar"
  5. Introducir el nombre de usuario y password de Sakai y hacer click en "OK"
  6. Aparecerá ahora una ventana en la pantalla del Macintosh que representa los recursos del sitio. Ahora es posible arrastrar y soltar archivos dentro de esta ventana y eliminar archivos al arrastrarlos a la Papelera.

Configurar WebDAV para Linux utilizando la Consola

Nota 1: Debe haber al menos un objeto (archivo, carpeta etc.) en el área de recursos del sitio antes de comenzar el proceso de configuración. Este proceso debe realizarse una vez para cada sitio, en Sakai antes de que se pueda usar WebDAV para transferir archivos a cualquier sitio, o viceversa.

Nota 2: Es necesario tener instalado en el sistema el driver WebDAV. En la mayoría de la distribuciones de Linux existe una versión precompilada. Si existe, simplemente tenemos que instalar el paquete correspondiente. Por ejemplo en Ubuntu se puede instalar escribiendo desde una terminal el comando:

sudo aptitude install davfs2

  1. Abrir un terminal en Linux.
  2. Ejecutar el siguiente comando, con un usuario con permisos de administrador:

    3. mount -t davfs server-url mountpoint -o uid=usuario,gid=grupo

    donde server-url es la URL para este sitio proporciona arriba y mountpoint es el directorio donde queremos montar la carpeta de recursos, uid es su nombre de usuario en su máquina local y gid el grupo al que pertenece el usuario.

    NOTA: el usuario introducido es el usuario de la máquina donde se está ejecutando el comando, no confundir con el usuario Sakai.

  3. Aparecerá un mensaje similar a:

    4. Please enter the username to authenticate with server
    https://x.x.x/dav/xxx
    hit enter for none
    Username:

    Introducir el nombre de usuario Sakai.

  4. A continuación de exigirá el password del usuario introducido con un mensaje similar a:

    5. Please enter the password to authenticate with server
    https://x.x.x/dav/xxx
    hit enter for none
    Username:

    Introduzca el password.

  5. Si todo ha funcionado correctamente, podremos acceder a la carpeta de recursos del sitio accediendo al directorio indicado al montar la carpeta.
